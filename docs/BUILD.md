# Building and Deploying the NativeCoop Module

## Source Location

**One source of truth:** `wse2work/Native-Coop-master/`

This is the ONLY module system directory. There is no other copy. All edits go here.

## Prerequisites

- Python 2.7 at `C:\Python27\python.exe` (the classic Warband module system does not run on Python 3)
- Mount & Blade: Warband installed (Steam), with the WSE2 rev-1145 binaries in the game directory
- A `Modules\NativeCoop\` directory in the game install
- `wse2work/Native-Coop-master/module_info.py` — set `export_dir` to your game's `Modules/NativeCoop/` path (this is where the build writes its output)

## Module Build (Full)

Always run a full build. Partial builds cause "Invalid Quick String ID" errors at runtime.

```cmd
cd wse2work\Native-Coop-master
build_module.bat
```

### Module Deploy

The build writes the compiled `.txt` files directly to `export_dir` (your game's `Modules/NativeCoop/` folder). No manual copy needed.

**Exception:** Resource files (`.brf`), `module.ini`, `Data/`, `SceneObj/`, `Music/`, `Sounds/`, `Textures/`, `Resource/` are NOT generated by the build. These are the module's static assets. If they are missing after a fresh install, copy them from `wse2work/Native-Coop-master/output/`.

## Deploy Configs and Launch Scripts

After building the module, copy the server configs and launch scripts into the game directory:

```bash
cp deploy/configs/*.txt             "<game>/Configs/"
cp deploy/configs/server_config.ini "<game>/"
cp deploy/scripts/*.bat             "<game>/"
cp deploy/coop.ini                  "<game>/"
```

`deploy/configs/` contains one `BattleServer_<n>.txt` per battle slot (0–3) plus `CampaignCoop.txt` for the campaign server.

## Launching

From the game directory, either start everything at once:

- `coop_launch_all.bat [N]` — campaign server + N battle-server slots (default 2, max 4) + a client

or start each process individually:

1. **Campaign server:** `coop_campaign_server.bat` (WSE2 dedicated campaign, port 7240)
2. **Battle servers:** `coop_battle_server_<n>.bat` for each slot n = 0–3 (slot n listens on port 7241 + 2n; the pool auto-adapts to however many slots are running)
3. **Client:** WSE2 launcher → select NativeCoop module → Multiplayer → join the campaign server

## Building the Client ASI (warband_coop.asi)

The client ASI injects the campaign server into the MP browser (Direct Connect from `coop.ini`), hooks native screen-close events, and writes the battle-server address into string register s59. It is loaded by the dinput8 ASI loader on the **client** (`mb_warband_wse2.exe`) only.

### Source Files

```
src/asi/
  coop.c                  # Browser hook, server injection, s59 writer
src/shared/
  hook.c/h                # Engine memory hooks
  modglobals.c/h          # Module-global (variables.txt) name resolution
  warband_addrs_wse2.c/h  # Engine address tables (client + dedicated)
```

### Build

```cmd
build\build.bat
```

(from bash: `cmd.exe //c build\\build.bat`)

The script auto-detects Visual Studio via `vswhere.exe`, sets up the x86 environment, and produces `bin\warband_coop.asi`.

---

## Building the Host Plugin (CoopWSEPlugin.dll)

The host plugin runs inside both dedicated-server binaries: in **COOP_HOST** mode (campaign server) it reads/writes module globals and hosts the ENet IPC listener; in **COOP_BATTLE** mode (battle server) it watches for battle end and signals the campaign server over IPC. The mode is detected from the exe name — no configuration needed.

### Prerequisites

- Visual Studio with the C++ desktop workload (any recent version — `vswhere.exe` auto-detects)
- x86 target (`vcvarsall.bat x86`), not x64 — Warband is a 32-bit game

### Source Files

```
src/
  plugin_main.cpp         # DllMain, plugin entry point, mode detection
  coop_campaign.c/h       # Campaign/battle server logic, IPC handlers, poll thread
  battle_ipc.h            # IPC packet framing (PKT_BATTLE_HELLO / PKT_BATTLE_END)
  wse_plugin.h            # WSE2 plugin API header
src/shared/
  battle_net.c/h          # ENet IPC link (battle server <-> campaign server, port 7242)
  wsedict.c/h             # .wsedict dict-file reader/writer
  modglobals.c/h          # Module-global name resolution + raw memory access
  hook.c/h                # Engine memory hooks
  crash_report.c/h        # Crash reporting + ring buffer
  warband_addrs_wse2.c/h  # Engine address tables (DED_*/CAMP_* for dedicated binaries)
enet/                     # ENet library source (vendored)
```

### Build

```cmd
build\build_plugin.bat
```

Produces `bin\CoopWSEPlugin.dll`.

## Building the Loader Proxy (winmm.dll)

Dedicated servers cannot use the dinput8 ASI loader, so the plugin is loaded by a `winmm.dll` proxy: it forwards all winmm exports to the real system DLL (saved as `winmm_sys.dll`) and loads `CoopWSEPlugin.dll` into processes whose exe name contains `dedicated`. Game clients ship the same proxy on disk but never load the plugin.

```cmd
build\build_loader.bat
```

Produces `bin\winmm.dll`.

## Deploy (C Artifacts)

Automatic — every successful C build ends with a call to `build\deploy.bat`, which:

- auto-detects the Warband install (Steam registry, every Steam library, common drive roots; set `GAMEDIR` to override)
- copies everything present in `bin\` (`warband_coop.asi`, `CoopWSEPlugin.dll`, `winmm.dll`) into the game directory
- creates `winmm_sys.dll` from the system winmm if missing
- installs `coop.ini` from `deploy/coop.ini` if it doesn't already exist

Run `build\deploy.bat` standalone to deploy without rebuilding. If the game or a server is running, the locked file is skipped with a WARN — close them and rerun.

## coop.ini

The plugin and client ASI read `coop.ini` from the game directory at startup (template: `deploy/coop.ini`):

```ini
[Coop]
Port=7240
HostIP=127.0.0.1
BattlePort=7241
Module=NativeCoop
```

There is no mode key — server role comes from the exe name. On each **client** machine, set `HostIP` to the campaign server's IP: battle joins build their address from it, and the `127.0.0.1` default silently bounces battle joins back to the MP menu.

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| Invalid Quick String ID | Partial module build | Run the full build |
| Module directory is invalid | Space in module name | Use `NativeCoop`, not `Native Coop` |
| UNABLE TO MAP GAME SCRIPT CODE | WSE2 scripts missing in module | Normal for vanilla-compatible scripts |
| Illegal Identifier: itm_cross_end | Missing Battle Time reference | Already fixed — stub definitions are in the module source |
| Invalid Scene_prop ID: -1 | Missing Battle Time scene prop | Already fixed — stub definitions are in the module source |

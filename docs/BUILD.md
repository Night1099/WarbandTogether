# Building and Deploying the NativeCoop Module

## Source Location

**One source of truth:** `wse2work/Native-Coop-master/`

This is the ONLY module system directory. There is no other copy. All edits go here.

## Prerequisites

- Python 2.7 at `C:\Python27\python.exe`
- Game installed at `F:\SteamLibrary\steamapps\common\MountBlade Warband\`
- Module directory `Modules\NativeCoop\` exists in game dir

## Deploy

`module_info.py` has `export_dir` pointing directly to the game's module folder:

```
F:/SteamLibrary/steamapps/common/MountBlade Warband/Modules/NativeCoop/
```

The build writes `.txt` files directly there. No manual copy needed.

**Exception:** Resource files (`.brf`), `module.ini`, `Data/`, `SceneObj/`, `Music/`, `Sounds/`, `Textures/`, `Resource/` are NOT generated by the build. These come from the module's static assets. If these are missing after a fresh install, copy them from `wse2work/Native-Coop-master/output/` or the original Native Coop distribution.

## Deploy Configs and Scripts

After building the module, deploy the server configs:

```bash
cp deploy/configs/BattleServer.txt  "<game>/Configs/"
cp deploy/configs/CampaignCoop.txt  "<game>/Configs/"
cp deploy/configs/server_config.ini "<game>/"
cp deploy/scripts/*.bat             "<game>/"
cp deploy/coop.ini                  "<game>/"
```

## Launching

1. **Battle server:** `coop_battle_server.bat` (WSE2 dedicated on port 7241)
2. **Campaign server:** `coop_campaign_server.bat` (WSE2 campaign dedicated on port 7240)
3. **Client:** WSE2 launcher → select NativeCoop module → Multiplayer → join campaign via LAN

## Building the Client ASI (warband_coop.asi)

The client ASI handles server browser injection, string register setup (s59), and auto-reconnect after battle. It's loaded by the ASI loader on the **client** (`mb_warband_wse2.exe` only).

### Source Files

```
src/asi/
  coop.c           # Browser hook, server injection, battle reconnect
src/shared/
  hook.c/h         # Engine memory hooks
```

### Build

```bash
cmd.exe //c "D:\\GIT\\M-BWarband-COOP\\build\\build.bat"
```

### Deploy

The build script deploys automatically: every successful build ends with a call to `build\deploy.bat`, which copies all artifacts present in `bin\` (ASI, plugin, winmm proxy) to the game directory.

**Note:** If the game or a server is running, the locked file is skipped with a WARN — close them and rerun `build\deploy.bat` standalone.

## Building the Host Plugin (CoopWSEPlugin.dll)

The host plugin handles campaign-to-battle IPC, ENet networking, and engine hooks. It's a C/C++ DLL loaded by the ASI loader on the **dedicated server**.

### Prerequisites

- Visual Studio with C++ desktop workload (any recent version — `vswhere.exe` auto-detects)
- `vcvarsall.bat x86` must be available (x86 target, not x64)

### Source Files

```
src/
  plugin_main.cpp      # DllMain, WSE2 plugin entry point
  coop_campaign.c/h    # Campaign-side logic (config, logging, encounter hooks)
  coop_battle.c/h      # Battle-side logic (state machine phases)
  battle_ipc.h         # Shared IPC structures
  wse_plugin.h         # WSE2 plugin API header
src/shared/
  coop_net.c/h         # ENet networking layer
  coop_protocol.h      # Network packet definitions
  hook.c/h             # Engine memory hooks
  state_snap.c/h       # State snapshot system
  crash_report.c/h     # Crash reporting + ring buffer
  warband_addrs.h      # Vanilla Warband engine addresses
  warband_addrs_wse2.h # WSE2 engine addresses
(root)
  kb.h                 # Knowledge base (RE'd struct offsets)
enet/                  # ENet library source (vendored)
```

### Build

```cmd
build\build_plugin.bat
```

This auto-detects Visual Studio, sets up x86 environment, compiles all C/C++ sources, links ENet, and produces `bin\CoopWSEPlugin.dll`.

From bash:
```bash
cmd.exe /c build\build_plugin.bat
```

### Deploy

Automatic — the build script ends with `build\deploy.bat`, which copies everything in `bin\` (`CoopWSEPlugin.dll`, `warband_coop.asi`, `winmm.dll`) to the game directory (`F:\SteamLibrary\steamapps\common\MountBlade Warband\`), creates `winmm_sys.dll` from the system copy if missing, and installs `coop.ini` if it doesn't already exist. Run `build\deploy.bat` standalone to deploy without rebuilding (set `GAMEDIR` to override the target).

### coop.ini

The plugin and client ASI read `coop.ini` from the game directory at startup (template: `deploy/coop.ini`):

```ini
[Coop]
Mode=host
Port=7240
HostIP=127.0.0.1
BattlePort=7241
DeltaInterval=5
DisableEdgeScroll=0
Module=NativeCoop
```

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| Invalid Quick String ID | Partial build | Run full build |
| Module directory is invalid | Space in module name | Use `NativeCoop` not `Native Coop` |
| UNABLE TO MAP GAME SCRIPT CODE | WSE2 scripts missing in module | Normal for vanilla-compatible scripts |
| Illegal Identifier: itm_cross_end | Missing Battle Time reference | Fixed in v0.4.3 — stub definitions added |
| Invalid Scene_prop ID: -1 | Missing scene prop from Battle Time | Fixed in v0.4.3 — stub definitions added |
